# feat/ — 特性方案文档

存放未在 MVP Sprint 主线内的**跨 Sprint 特性方案** / **架构改造** / **专题设计**。每个特性一个 markdown，自带：

1. 背景 / 痛点
2. 方案设计（含对比、架构图）
3. 时间预算（人天 + 周）
4. 分阶段（P0/P1/P2/P3）实施 + Story 拆分
5. 跟踪表（状态列每周一更新）
6. 与 MVP Sprint 的关系
7. 风险 + 缓解

> 与 `planning/` 的区别：planning 是 sprint 节奏的整体规划；feat 是单个特性的深度方案，可能跨多个 sprint 或独立排期。
