# 后台管理系统 PRD — 管理员 & 店家门户

**版本：** v0.1 草案  
**日期：** 2026-05-10  
**状态：** 待审核

---

## 1. 背景与目标

在现有 C 端商城（用户下单、购物车、支付、评论等）的基础上，建设 B 端后台管理系统，服务两类角色：

| 角色 | 定位 |
|------|------|
| **平台管理员** | 负责平台整体运营：审核店家、处理投诉、配置平台活动、监控风险 |
| **店家（商户）** | 负责自有店铺运营：商品管理、订单履约、客服、结算、活动 |

系统目标：
- 降低平台运营的人工成本
- 给店家提供自助化的店铺管理工具
- 支撑活动运营和风控闭环

---

## 2. 角色与权限

```
超级管理员
  └─ 平台管理员（多名，按模块分权）
       └─ 审核员、运营、客服、财务（细粒度权限组）

店家
  └─ 主账号（店主）
       └─ 子账号（员工：客服、仓管、财务等，权限可配置）
```

## 3. 管理员功能模块

### 3.1 店家入驻管理

**功能点：**
- 查看入驻申请列表（待审核 / 审核中 / 已通过 / 已拒绝）
- 审核申请详情：营业执照、法人身份证、店铺信息、品类资质
- 审核操作：通过（自动开通店家账号）/ 拒绝（填写原因，通知申请人）/ 补充材料
- 店家等级晋升申请审核（店家主动发起，见 §4.6）
- 店家注销/暂停申请审核

**数据流：** `mall-shop-rpc` + `mall-workflow-rpc`（审核流）

---

### 3.2 店家投诉处理

**功能点：**
- 投诉工单列表（来源：用户投诉店家、店家投诉用户）
- 工单详情：投诉内容、证据图片、关联订单
- 处理操作：
  - 驳回投诉（填写理由）
  - 支持投诉方（触发相应惩罚或退款）
  - 调解（双方达成一致后关闭）
- 投诉时效提醒（SLA：24h 内响应，72h 内结案）
- 投诉统计：店家被投诉频率、用户投诉频率

**数据流：** `mall-risk-rpc` + `mall-order-rpc` + `mall-review-rpc`

---

### 3.3 店家奖励 / 惩罚

**功能点：**

| 动作 | 触发场景 | 效果 |
|------|---------|------|
| 加分 | 优质服务、活动参与积极 | 信用分 +N |
| 扣分 | 投诉坐实、违规商品、虚假发货 | 信用分 -N |
| 警告 | 轻微违规 | 站内信通知 |
| 限制功能 | 连续违规 | 禁止上新品、禁止参加活动 |
| 封禁店铺 | 严重违规 | 店铺下线，商品隐藏 |
| 解封 | 整改完成 | 恢复运营 |
| 奖励激励 | 月度优秀商家 | 流量扶持标签、信用分加分 |

- 操作需填写理由并记录日志（不可删除，供审计）
- 分值达到阈值自动触发规则（由 `mall-rule-rpc` 配置）

**数据流：** `mall-shop-rpc` + `mall-rule-rpc` + `mall-reward-rpc`

---

### 3.4 平台活动 / 优惠配置

**功能点：**
- **限时秒杀**：设置活动时间段、商品池、最低折扣比例、单用户限购
- **平台满减**：全平台 / 指定品类满 X 减 Y，支持叠加/互斥规则
- **平台优惠券**：发券（定向用户群 / 全员）、券模板、有效期、领取上限
- **节日主题活动**：首页 Banner 关联、活动专题页配置（元数据，实际渲染由前端读取）
- **积分兑换活动**：用积分兑换商品或优惠券
- 活动列表：进行中 / 即将开始 / 已结束
- 活动数据看板：参与人数、GMV 增量、优惠总额
- 活动和优惠规则需要支持低代码模式，也就是通过组件，拖拽的方式，用户可以很友好，低成本的配置规则，活动规则包括：
  - 条件，例如，活跃度/签到，下单流水达到多少等；活动预算总金额；
  - 剔除条件，例如黑名单（或者风控标识危险用户等）/ 节假日签到不算 / 风控标识刷单流水不算等；
  - 奖励，例如，经验，信誉分，现金等；
  

**数据流：** `mall-activity-rpc` + `mall-rule-rpc` + `mall-reward-rpc`

---

### 3.5 用户管理（辅助模块）

**功能点：**
- 用户列表：查询、封禁/解封、查看订单历史
- 用户投诉受理（与 §3.2 共用工单系统）

---

### 3.6 内容审核

**功能点：**
- 商品上架自动审核（图片违规检测 + 关键词过滤）
- 人工审核队列（机器打标需人工复核的商品）
- 评论违规举报处理
- 审核规则配置（敏感词库、图片分类阈值）

**数据流：** `mall-product-rpc` + `mall-review-rpc` + `mall-risk-rpc`

---

### 3.7 数据统计 & 报表

**功能点：**
- GMV、订单量、DAU、新增店家、新增用户（日/周/月）
- 退款率、投诉率、好评率趋势
- 活动效果对比报表
- 数据导出（CSV/Excel）

## 4. 店家功能模块

### 4.1 商品管理

**功能点：**
- **商品列表**：搜索、筛选（在售/下架/审核中/违规）、批量操作
- **发布商品**：
  - 基础信息：名称、品类、品牌、主图（最多 9 张）、视频
  - 详情页：富文本编辑器（图文混排）+ 规格参数表格
  - 规格 SKU：多维度规格（颜色×尺寸），每个 SKU 独立定价、库存、条码
  - 物流：重量、体积、运费模板
- **编辑商品详情**：修改内容后重新进入审核队列
- **上线 / 下线**：手动控制，下线不删除数据
- **库存管理**：直接修改库存，记录变更日志
- **价格调整**：支持批量改价

**数据流：** `mall-product-rpc` + 内容审核

---

### 4.2 订单管理 & 发货

**功能点：**
- **订单列表**：按状态筛选（待付款 / 待发货 / 已发货 / 已完成 / 退款中）
- **订单详情**：商品明细、买家信息、地址、备注
- **发货操作**：
  - 填写快递公司 + 运单号（手动）
  - 批量发货（CSV 导入运单号）
  - 集成电子面单（未来扩展）
- **退款处理**：
  - 用户申请退款后，店家确认 / 拒绝退款
  - 拒绝需填理由，平台可介入仲裁
- **物流跟踪**：展示快递轨迹（接 `mall-logistics-rpc`）

**数据流：** `mall-order-rpc` + `mall-logistics-rpc` + `mall-payment-rpc`

---

### 4.3 评论管理（回复评论）

**功能点：**
- 评论列表：按商品、星级、时间筛选
- 回复评论（一条评论只能回复一次，可编辑 24h 内）
- 申请删除违规评论（提交给平台审核）
- 评论统计：平均分、好评率、关键词词云

**数据流：** `mall-review-rpc`

---

### 4.4 客服设置

**功能点：**
- 在线客服（IM）工作台（此 PRD 定义接口规范，IM 服务独立）
- 自动回复配置：关键词触发 → 预设话术
- 客服账号管理：邀请员工、分配接待权限
- 工作时间设置：非工作时间展示自动回复
- 客服绩效：接待量、响应时长、满意度评分

**数据流：** 新增 `mall-im-rpc`（待规划）+ `mall-shop-rpc`

---

### 4.5 收款 & 结算

**功能点：**
- **账户余额**：可提现余额、冻结金额（退款预留）、累计到账
- **提现申请**：填写银行账户 / 支付宝，金额校验（最低提现额、手续费说明）
- **账单流水**：收入明细、退款扣款、平台服务费、提现记录
- **结算周期**：T+N 结算配置（由平台设置，店家只读）
- **发票管理**：申请平台开具服务费发票

**数据流：** `mall-payment-rpc` + 新增结算账户模型

---

### 4.6 等级申请

**功能点：**
- 当前等级及权益展示（流量扶持系数、活动资格、佣金费率）
- 等级晋升条件展示（GMV 门槛、好评率、经营月数、信用分）
- 一键申请晋升（满足条件后解锁）
- 申请状态跟踪（提交 → 平台审核 → 通过/拒绝）
- 等级降级预警通知

**数据流：** `mall-shop-rpc` + `mall-rule-rpc` + `mall-workflow-rpc`

---

### 4.7 店铺活动 / 优惠设置

**功能点：**
- **店铺优惠券**：创建券（满减/折扣/直减）、设置发放量、有效期
- **店铺满减**：满 X 减 Y，支持多梯度
- **商品限时折扣**：指定商品 SKU 设置折扣时段
- **参加平台活动**：报名平台限时秒杀（需填写活动价，平台审核后生效）
- **活动日历**：已创建 / 即将开始 / 进行中 / 已结束

**数据流：** `mall-activity-rpc` + `mall-rule-rpc`

---

### 4.8 店铺数据看板

**功能点：**
- 今日/本周/本月：销售额、订单量、访客数、转化率
- 商品排行：销量 TOP10、退款率 TOP10
- 流量来源分析
- 与行业均值对比（匿名基准）

## 5. 系统公共能力

| 能力 | 说明 |
|------|------|
| 权限控制 | RBAC，管理员按模块授权，店家子账号按功能授权 |
| 操作日志 | 所有写操作记录操作人、时间、内容快照，不可删除 |
| 消息通知 | 站内信 + 邮件 + 短信（审核结果、订单状态、投诉通知） |
| 文件存储 | 商品图片、资质文件统一走 MinIO |
| 搜索 | 商品/订单全文检索（复用 ElasticSearch 或 ProxySQL FTS） |
| 导出 | 订单、账单、商品列表支持 CSV/Excel 异步导出 |

## 7. 技术约束 & 边界说明

1. **API 路由分离**：后台 API 统一走 `/admin/v1/...` 和 `/merchant/v1/...`，与 C 端 `/api/v1/...` 隔离，在 `mall-api` 中路由。
2. **IM 服务**：`mall-im-rpc` 为新增服务，本期 PRD 仅定义接口契约，实现在后续迭代。
3. **富文本存储**：商品详情页内容存为 HTML 或 JSON（Quill delta），图片走 MinIO，不内嵌 base64。
4. **异步任务**：结算、活动数据统计、CSV 导出均走 `mall-activity-async-worker` 或新增 worker，不阻塞请求。
5. **现有 RPC 扩展**：各 Epic 涉及已有 rpc 时，优先在原 rpc 中扩展 proto 接口，不新建冗余服务。
