## 🔧 主要交付

### S3.1/S3.2 退货换货状态机

**mall-order-rpc**：
- `refund_request` 表新加 7 列：`refund_type / return_tracking_no / return_carrier / return_ship_time / return_received_time / return_inspection_passed / exchange_new_order_id`
- 3 个新 RPC：
  - `UserShipReturn(refund_id, tracking_no, carrier)` —— 用户填寄回单号
  - `MerchantInspectReturn(refund_id, passed, remark)` —— 店家验货；如 type=2 验过自动触发 ExecuteRefund
  - `MerchantShipExchange(refund_id, tracking_no, carrier)` —— 换货发新单（复制原订单）
- 退款类型：1=refund_only / 2=return_refund / 3=exchange

### S3.3 退货物流跟踪

- `mall-api` 加 `POST /api/refund/:id/ship-return`
- 简化：直接存 tracking_no + carrier，不接快递商 API（参考 S6 之后再接）

### S3.4 平台账务台账

**mall-payment-rpc**：
- 新表 `account_ledger`：direction(1=credit/2=debit) + category(order_income/refund/commission/withdrawal/adjustment) + amount + running_balance + 关联 order_id/refund_id/withdrawal_id + ref_no
- `ConfirmMockPay` (S1) + `ExecuteRefund` (S2) 事务内追加 INSERT ledger
- 新 RPC：`ListLedger` / `GetShopLedgerSummary`

### S3.5 对账机制

- 新 RPC `RunReconciliation(shopId)`：比对 ledger 累积净额 vs wallet (balance + frozen)
- 失败时返回 `ShopReconcileResult{ledger_credit, ledger_debit, ledger_net, wallet_total, diff, passed}`
- e2e 验证：order_income 20000 + refund -20000 = 0, wallet=0, **diff=0 passed:true**

### S3.6 admin 后台

**yw-mall-admin**（API 网关）：3 个新路由
- `GET /admin/v1/ledger` 流水分页
- `GET /admin/v1/ledger/summary` 汇总卡片
- `POST /admin/v1/ledger/reconcile` 触发对账

**yw-mall-admin-fe**：`src/views/ledger/index.vue` —— 财务台账模块（菜单组「财务」下）

## 🐛 修的 2 个坑

### 1. SubmitRefundRequest INSERT 漏写 refund_type 列

executor 改了 proto 加 refund_type 字段，也加了 DB 列，但 INSERT SQL 没改，导致 type 永远存为 DEFAULT 1。

修：INSERT 加上 `refund_type` 列 + 用 `refundTypeOrDefault()` helper 兜底 0 → 1。

### 2. mall-api types.SubmitRefundReq 缺 RefundType 字段

C 端 → mall-api → mall-order-rpc 链路传 refundType 时，mall-api 层 types struct 不接受这字段。

修：types + logic 双向加 `RefundType int32 json:"refundType,optional"`。
