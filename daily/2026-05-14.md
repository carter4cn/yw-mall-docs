## 🔐 登录改造 P0（feat/login-revamp.md）

**触发动因：** S1-S3 期间反复遇 JWT "signature is invalid"，根因是 user-rpc `onConfigChange` 中 `yaml.Unmarshal` 静默丢 `JwtAuth.AccessSecret` 嵌套字段；e2e 全程靠 `openssl dgst -sha256 -hmac` 手工签 token 兜底。**决策**：不修 JWT，整体替换。

### 架构：Opaque Token + Redis Session

- `access_token` = base64url(32 字节随机) — TTL 1800s
- `refresh_token` = base64url(32 字节随机) — TTL 604800s，每次 refresh 旋转
- `csrf_token` 随 session 下发，写操作走 X-CSRF-Token 双重提交
- `user_sessions:{uid}` SET 索引所有活跃 access_token，改密/踢出一键清

## 🔐 P0.5 — admin-api SessionMiddleware 切换

> P0 收尾后发现 yw-mall-admin 仍走 JWT，本日内追加完成切换。

### 扩展 session 携带 perms（admin RBAC 需要）

- proto: `SessionInfo` + `CreateSessionReq` 加 `repeated string perms`
- sessionhelpers.go: `sessionPayload` / `refreshPayload` 加 `Perms []string`
- createsession / validate / refresh logic: 全链路携带 perms

### yw-mall-admin: JWT 中间件整体替换

- `middleware/jwt.go` → 只保留 `Claims` + `ClaimsFromContext` + `withClaims` + `extractBearer`；删 IssueToken / ParseToken / golang-jwt 依赖
- `middleware/rbac.go` → `SessionAuthMiddleware{userRpc, requiredRole}`：调 `UserRpc.ValidateSession`，role 不匹配返 403
- `logic/auth_logic.go` → `AdminLogin` / `MerchantLogin` 改走 `UserRpc.CreateSession`；新增 `AdminRefresh` / `MerchantRefresh` / `AdminLogout` / `MerchantLogout`
- 新路由：`POST /admin/v1/refresh`（public）+ `POST /admin/v1/logout`（protected）+ merchant 对称两个
- `svc/servicecontext.go` 去 `JwtSecret` HotConfig
- `go mod tidy` → `golang-jwt/jwt/v4` 降级为 indirect

### yw-mall-admin-fe `admin/` SPA 同步

- `stores/user.ts`：`accessToken / refreshToken / csrfToken / expiresAt` + `setSession()` + async `logout()` 调后端
- `api/request.ts`：单飞 `/admin/v1/refresh` + 401 重试 1 次 + 写操作注入 X-CSRF-Token
- `api/auth.ts` + `types/api.ts` + `layouts/AdminLayout.vue` 配套
- `merchant/` SPA 仍为占位（README），后端 `/merchant/v1/*` 接口就位待对接
