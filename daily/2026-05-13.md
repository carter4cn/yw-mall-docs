## 🐛 修的 3 个 go-zero sqlx 坑

### 1. 匿名 struct 不能用作 QueryRowCtx 目标

```go
// ❌ 失败：not matching destination to scan
var row struct {
    Id int64 `db:"id"`
    // ...
}
svc.SqlConn.QueryRowCtx(ctx, &row, "SELECT id, ... FROM ...")

// ✅ 工作：包级命名类型
type orderRowForRefund struct {
    Id int64 `db:"id"`
    // ...
}
var row orderRowForRefund
```

go-zero `sqlx` 内部 `unmarshal` 通过 reflect 看类型的 PkgPath，匿名 struct 路径为空，跳过 db tag 解析。

### 2. `sql.NullInt64` 不能作 QueryRowCtx 目标

```go
// ❌ 失败：not matching destination to scan
var existingSum sql.NullInt64
svc.SqlConn.QueryRowCtx(ctx, &existingSum, "SELECT IFNULL(SUM(...), 0) FROM ...")

// ✅ 工作：plain int64（IFNULL 保证不为 NULL）
var existingSum int64
```

### 3. `bill_record` 缺 refund_no 列

executor 写 INSERT 时引用了不存在的列。补迁移：
```sql
ALTER TABLE bill_record ADD COLUMN refund_no varchar(64) NOT NULL DEFAULT '';
ALTER TABLE bill_record ADD INDEX idx_refund_no(refund_no);
```
