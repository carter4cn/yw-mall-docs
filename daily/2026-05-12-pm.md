### 2. ⚡ cancel loop 时区坑 — 关键修复

**症状**：刚创建的待支付订单 20-60 秒后就被自动取消，cancel_reason='auto:expired'。

**根因**：cancel.go 用 Go `time.Time` 计算 cutoff（`now - 15min`），通过 go-sql-driver 写入 MySQL TIMESTAMP 比较：
- worker 容器 TZ=Asia/Shanghai (UTC+8)
- MySQL 容器 session_tz 默认 UTC
- MySQL 列 create_time 用 UTC 存储
- Go time.Time 通过 driver 转换格式时，与 MySQL 的 session timezone 解释不一致 → 实际比较的时点漂移 8h

**修法**：改用纯 SQL `UNIX_TIMESTAMP(create_time) + ? < UNIX_TIMESTAMP()` 做比较，全部转秒数，绕开 Go time.Time ↔ MySQL TIMESTAMP 的 TZ 桥接。

```diff
-WHERE status = 0 AND create_time <= ?  -- 传 Go time.Time，TZ 漂移
+WHERE status = 0 AND UNIX_TIMESTAMP(create_time) + ? < UNIX_TIMESTAMP()  -- 全 SQL，绕开 TZ
```

后续涉及"过去 N 秒/分钟"判断的查询都用这个模式。
