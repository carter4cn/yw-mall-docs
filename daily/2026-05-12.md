### 踩到的坑

### 1. `review_admin.sql` 用了 MariaDB-only 语法

```sql
ALTER TABLE `review` ADD COLUMN IF NOT EXISTS `shop_id` bigint NOT NULL DEFAULT 0;
CREATE INDEX IF NOT EXISTS `idx_shop_id` ON `review`(`shop_id`, `status`, `id`);
```

MySQL 8 不支持 ALTER 和 CREATE INDEX 上的 `IF NOT EXISTS`，导致 P1 上线时 `review_delete_request` 表静默没建出来。**修法**：拆成普通 `ALTER TABLE ... ADD COLUMN` + `ADD INDEX`；今后所有 DDL 走 plain 语法，幂等性靠 db-init 流程外层保证。

### 2. F-5 测试时读不到刚写的 credit_score

```go
// 旧版
UPDATE shop SET credit_score = GREATEST(0, LEAST(200, credit_score+?)) WHERE id=?
SELECT credit_score FROM shop WHERE id = ?
```

UPDATE 走 ProxySQL 主，SELECT 走从库，复制还没追上 → 读到旧值，阈值 switch 永远命中 default 分支，看起来"没生效"。

**修法**：read-then-write（一次 SELECT + 进程内计算 next + 一次 UPDATE），把 next 直接传给 `applyThresholdRules`，**永远不再回库读最新值**。

### 关键 bug fix：`parseId` 吞掉请求 body

**症状**：所有 `:id` + JSON body 的 admin 路由（`POST /admin/v1/shops/:id/credit`、`/shops/:id/status` 等）返回 `400 Bad Request`，body `string: ``, error: EOF`。

**根因**：handler 写法

```go
func adjustCreditScoreHandler(...) http.HandlerFunc {
    return func(w, r) {
        id, _ := parseId(r)                                // 1️⃣
        var req types.AdjustCreditScoreReq
        if err := httpx.Parse(r, &req); err != nil { ... } // 2️⃣ EOF here
    }
}
```

`parseId` 内部 `httpx.Parse(r, &idPath{})` 会**读取并消费 body**（go-zero httpx 总是 decode body，不管目标 struct 有没有 json tag），到第 2 步再 Parse 时 body 已经 EOF。`OpLog` middleware 虽然 `io.NopCloser` 还原过一次 body，但 `parseId` 在 middleware 之后又消费了一次。

**修法**：换成 `pathvar.Vars(r)["id"]` + `strconv.ParseInt`，**不碰 body**：

```go
func parseId(r *http.Request) (int64, error) {
    raw, ok := pathvar.Vars(r)["id"]
    if !ok { return 0, nil }
    return strconv.ParseInt(raw, 10, 64)
}
```

`parseRid`（`:rid` 二级路径参数）同步修。影响所有 13 个使用 `parseId` 的 handler，但都是兼容修复（只是少读了一次 body）。
