### 今天的核心修复

### `mall-common/configcenter/loader.go`：yaml.Unmarshal → conf.LoadFromYamlBytes

**症状**：所有 RPC 启动后 `Etcd.Hosts` 为空，导致服务**不向 etcd 注册自己**，
mall-api 通过 etcd discovery 找不到任何 RPC，调用全挂。

**根因**：`yaml.v3` 在多层匿名嵌入结构上**静默丢字段**：

```
Config (用户)
  └── service.ServiceConf  (anonymous embed, go-zero)
        └── discov.EtcdConf
              └── Hosts []string  ← 丢
```

`yaml.Unmarshal` 解析进 `dest any` 时只填了顶层字段；嵌套层的 `Hosts/Key`
等字段没传下去。这同样影响 `RestConf.Host`、`RestConf.MinIO.Endpoint` 等
go-zero 内部 embed 的字段。

**修法**：换成 go-zero 自家的 `conf.LoadFromYamlBytes`，它认识嵌入结构和
`json:",default=..."` 标签，按 `mapping → reflect.Set` 路径正确赋值。

附带 `go mod tidy`：
- `mall-common/go.mod` 移除 `gopkg.in/yaml.v3` 直接依赖
- `mall-review-rpc/go.mod`、`mall-activity-async-worker/go.{mod,sum}` 同步
- `mall-activity-async-worker/go.mod` 新增 indirect: `pelletier/go-toml/v2`、`titanous/json5`、`yaml.v2`（go-zero conf 间接依赖）

### 今天的核心改动

### 1. `scripts/rebuild.sh` — 全新并行重建脚本（替换原 `compose build`）

| 特性 | 说明 |
|---|---|
| 并行 | `JOBS=4` 个 service 同时 build；信号量目录控制并发 |
| 进度条 | 5 秒刷新一次，DONE / TOTAL / 已用时间 |
| 增量模式 | 默认 `NO_CACHE=1`（不变）；设 `NO_CACHE=0` 命中 go.mod 缓存层，5–10× 加速 |
| 共享 cache | BuildKit `--mount=type=cache` 共享 GOPATH 与 build cache，4 个并发不再各拉一份依赖 |
| 国内代理 | `GOPROXY=https://goproxy.cn,direct` 透传给 `go mod download` |
| 块状 replace | go.mod 的 `replace ( ... )` 形式正确解析（旧版漏抓导致 mall-api / mall-activity-rpc 直接挂） |
| 失败保留日志 | trap 在 FAIL>0 时保留 LOG_DIR，否则清掉 |
| source guard | 被 `source` 时立刻 return；防止 OMC 之类工具误触发真实 build |

**用法**：

```bash
make rebuild                                                      # 全量 no-cache（不变）
NO_CACHE=0 make rebuild                                            # 增量 + cache mount
NO_CACHE=0 GOPROXY=https://goproxy.cn,direct make rebuild         # 国内全速（推荐日常）
```

### 2. `scripts/config-push.sh` — 容器名替换

`transform_for_deploy` 函数推 etcd 前把本地 `127.0.0.1:<port>` 替换成
对应容器名：etcd1 / proxysql / redis-master / minio / kafka1-3 / dtm /
mall-api。否则容器内服务把 127.0.0.1 当 loopback，互相找不到。

### 3. `seed.Dockerfile` — `mkdir -p /out`

`go build -o /out/seed-$svc` 之前缺 `mkdir`，第一次 RUN 直接报
"no such file or directory"。

### 4. `Makefile`

### Podman Desktop 看不到容器

`systemctl --user enable --now podman.socket`，socket 在
`/run/user/1000/podman/podman.sock`。Desktop 重启后能识别。

- `--no-cache` 全量 rebuild 在我这条网络上 ≈ 60 分钟，**日常迭代必须用

  镜像没变）。如果只想换 fe 不动后端，最好不要走 rebuild 全流程。
